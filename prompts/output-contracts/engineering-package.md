# Output Contract: Engineering Package

Every Daedalus engineering package should follow this structure.

## Metadata

- Project name:
- Request date:
- Requestor:
- Environment:
- Status:
- Approval required: Yes

## 1. Requirement Interpretation

Explain what the user is asking for in engineering terms.

## 2. Assumptions and Constraints

List assumptions and constraints.

## 3. Proposed Architecture

Describe the recommended architecture.

## 4. Security Considerations

List security concerns and controls.

## 5. Dependencies

List dependencies.

## 6. Implementation Phases

Break the project into phases.

## 7. Generated Artifacts

List expected files and artifacts.

## 8. Validation Procedure

Provide validation checklist.

## 9. Rollback Procedure

Provide rollback steps.

## 10. Approval Gates

List required approvals.

## 11. Documentation Updates

List required documentation.

## 12. Project-Management Tasks

Provide tasks in a ClickUp-compatible hierarchy.
