# Daedalus v0.2.0 Release Notes

## Release Summary

Daedalus v0.2.0 establishes the first usable MVP baseline of the Daedalus engineering platform.

This release includes the operating model, project memory, workflows, templates, schemas, validation tooling, GitHub governance, architecture diagrams, local CLI, tests, CI validation, portfolio documentation, MVP closeout, and the first real lab-oriented engineering package.

## Safety Model

This release preserves the core Daedalus safety model:

- Proposed work by default
- Human approval before infrastructure-impacting action
- No automatic infrastructure execution
- No plaintext secrets
- Validation required
- Rollback required when applicable
- Security review required for access, identity, exposure, backups, and privileged automation
- ADRs required for major decisions

## Validation Expectations

Before tagging this release, run:

```powershell
python scripts/validate_repo.py
python scripts/validate_schemas.py
python -m pytest
python -m daedalus.cli init-check
```

GitHub Actions should also pass.

## Tagging

After validation passes:

```powershell
git tag v0.2.0
git push origin v0.2.0
```
